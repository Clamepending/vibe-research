- [code-review-prompt-tuning](experiments/code-review-prompt-tuning/README.md) — best system prompt for `claude-sonnet-4-6` to produce substantive code review on a buggy Python payment-validator function. Winner: V2 `variants/v2_checklist.txt` (named-category list) at `/Users/mark/Desktop/projects/code-review-prompt-tuning` commit `1d52fa3`, tag `exp/code-review/best`. Key findings: (a) baseline is already substantive — the "add type hints / consider error handling" failure mode did not occur in 16 samples; (b) category priming (V2) hits 6/6 ground-truth issues every sample vs baseline 4.75/6; (c) persona framing (V1) adds concurrency/TOCTOU catches but doesn't broaden core coverage; (d) adversarial "find subtle bugs" framing paradoxically *reduces* coverage.
- [uncertainty-admission-prompt-tuning](experiments/uncertainty-admission-prompt-tuning/README.md) — can prompt-craft make `claude-sonnet-4-6` more reliably admit "I don't know" vs confabulate? Baseline already 90% correct (12/12 fabricated essays rejected, 9/9 misconceptions addressed, 9/9 fake APIs rejected). Winner `v6_focused` at `/Users/mark/Desktop/projects/uncertainty-admission-prompt-tuning` commit `78d2298`, tag `exp/uncertainty/best`, scored 60/60 on two independent n=3 runs (120 total, 0 confabulations). Key finding: the residual failure mode on frontier models isn't "makes up fake APIs" but "asserts a specific number from recollected gist" (francium melting point stated as measured not extrapolated; Scoraig census figures invented). A numeric-fact epistemic-status clause closes that gap. Generic "it's OK to say IDK" only helps marginally.
